# 事故案例库结构与边界

## 核心定位

事故案例库是企业落地的核心资料，不是可选项。案例用于警示教育、相似性参考、风险链路复盘、报告引用和双重预防措施补强。不能因为相似案例就自动判定当前矿井必然发生事故，不能替代现场复核、人工审核和调度处置。

## 字段结构

| 字段 | 说明 |
| --- | --- |
| `case_id` | 案例唯一编号 |
| `事故名称` | 官方事故名称或规范化名称 |
| `事故时间` | 事故发生时间 |
| `事故地点` | 省、市、矿井或工作面位置 |
| `事故类型` | 事故调查中的事故类型 |
| `灾害类型` | 煤与瓦斯突出、瓦斯爆炸、瓦斯超限等 |
| `伤亡情况` | 死亡、受伤、涉险人数 |
| `直接经济损失` | 官方报告载明金额 |
| `矿井类型` | 高瓦斯、突出矿井、低瓦斯等 |
| `工作面/巷道位置` | 采掘工作面、回风巷、抽采巷等 |
| `事故经过` | 过程摘要 |
| `直接原因` | 官方直接原因 |
| `间接原因` | 官方间接原因 |
| `管理原因` | 管理缺陷 |
| `技术原因` | 技术、系统、设备缺陷 |
| `违章行为` | 违章指挥、违章作业、违反规程 |
| `失效措施` | 失效或缺失的防控措施 |
| `缺失监测指标` | 缺失或未有效使用的监测指标 |
| `前兆信号` | 瓦斯、风速、压力、钻屑量、微震等前兆 |
| `涉及法规条款` | L1 资料 source_id 或条款 |
| `涉及标准规范` | L2 资料 source_id 或条款 |
| `整改措施` | 官方整改或建议措施 |
| `警示教育要点` | 培训和警示要点 |
| `可映射知识卡` | 后续知识卡候选 |
| `可映射风险链路` | 风险链路节点 |
| `是否适合训练/评估` | 仅用于评估或教学时为 true |
| `是否脱敏` | 是否已完成脱敏 |
| `来源等级` | 通常为 L3 |
| `来源链接/文件` | 官方链接或私有存储路径 |
| `人工审核状态` | pending / approved / rejected / needs_redaction |

## 误用边界

案例相似性只能提示需要复核，不能输出必然发生。案例原因必须标注来自官方报告、监管通报或人工审核摘要。案例不能替代法规条文、标准要求、现场调度研判和人工审核。涉及企业内部资料、人员姓名、精确位置、设备编号时必须脱敏。
