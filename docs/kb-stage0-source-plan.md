# 阶段零资料源库建设计划

## 阶段零目标

阶段零只建立企业级资料源库的底座，不做前端页面、不做最终 AI 问答、不训练模型。目标是让每一份资料在入库前都能回答来源可靠性、证据等级、全文保存边界、AI 引用边界、合规依据边界、业务模块映射、人工审核、脱敏、风险/措施/隐患/报告/案例映射，以及教育研究用途边界。

## 资料库边界

资料源库记录法规、标准、政策、事故案例、企业制度、现场资料、设备资料、中文论文、英文论文、数据样本和模型评估资料。它不是公开全文仓库，也不是自动执法或自动事故判定系统。

全文可以在合法教育研究授权范围内下载，但全文只进入 `local-kb/raw/**`、`local-kb/downloads/**`、`local-kb/extracted/**` 或 Supabase 私有 Storage。GitHub 只提交元数据、清单、脚本、报告、结构化摘要和规则文件。

## 企业落地用途

- `monitoring_warning`：监测预警、报警、断电、复电、阈值和测点边界。
- `source_tracing`：溯源研判、风险链路复盘、异常解释。
- `dual_prevention`：风险分级管控、隐患排查治理、措施补强。
- `digital_twin`：空间对象、测点位置、通风和抽采系统资料绑定。
- `data_model`：数据字段、63 个通道来源、质量评估、模型验证。
- `knowledge_intelligence`：后续 Supabase 知识库和 RAG 引用来源。
- `report_generation`：报告依据、案例引用、整改建议来源。
- `system_management`：资料权限、审核、脱敏、下载日志、版本管理。

## 后续阶段调用方式

后续 Supabase 知识库先读取 `kb_source_registry` 和分类型登记表，再基于已审核、可引用、可存储的资料生成知识卡、切片和向量索引。RAG 检索必须保留 `source_id`、证据等级、来源链接、是否可合规引用和人工审核状态。双重预防、监测预警、溯源研判、数字孪生、数据模型和报告生成只能引用符合边界的资料，不能把论文结论写成法规要求，不能用事故案例替代现场复核。

## 阶段零交付物

资料分类、证据等级、CNKI/WoS 检索策略、事故案例库字段、入库规则、存储策略、source manifest、校验/去重/报告脚本和 Supabase 阶段零资料登记表。
