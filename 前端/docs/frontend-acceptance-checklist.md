# 前端验收清单

当前阶段：后端展示闭环与通道口径修正 / 阶段 3：验收收口与演示整理。
阶段 9 视觉精修只作为历史阶段保留；当前验收重点是预警后端展示闭环、通道口径、写操作边界和演示路径。

## 1. 结构

- [ ] 没有重新改导航、入口和路由层级。
- [ ] 没有删除既有路由。
- [ ] 仍使用 `frontend/src/app` 作为唯一 App Router 路由根。

## 2. 业务表达

- [ ] 总览页能回答：现在安全吗？风险在哪里？哪些传感器触发？当前值和阈值是多少？谁在处理？处理到哪一步？下一步点哪里？
- [ ] 关键数据有单位、阈值、状态和更新时间。
- [ ] mock 边界没有丢失，mock 数据没有伪装成真实生产结论。
- [ ] 系统管理、日志、配置页可读性优先。

## 3. UI 质量

- [ ] 当前指标卡样式被保留并统一。
- [ ] 页面更接近参考图。
- [ ] PSD / 预览图组件被转译为 UI 语言，而不是整图贴图。
- [ ] 中心主视觉更像煤矿生产态势。
- [ ] 风险颜色统一：低风险蓝、一般风险黄、较大风险橙、重大风险红。
- [ ] 粒子、光效、动画不遮挡内容。

## 4. 验证

- [ ] 文档任务已运行 `git status --short`、`git diff --name-only`、`git diff --check`。
- [ ] 源码任务按风险运行必要 `lint` / `build`。
- [ ] 重大视觉改动只截图受影响页面，检查拥挤、重叠、文字不可读和内容遮挡。

## 5. 提交

- [ ] 没有提交 PSD、rar、zip、参考图、大型素材、`node_modules`、`.next` 或 `.env`。
- [ ] `docs/frontend-progress.md` 已简洁记录本次任务。
- [ ] 已 commit / push，或已说明失败原因。

## 6. 后端展示闭环验收

- [ ] `/monitoring` 能展示后端健康、动态数据量、节点数、最新数据时间。
- [ ] `/monitoring/realtime` 能展示趋势曲线和近期采样。
- [ ] `/monitoring/channels` 能展示通道元数据、单位、位置、来源。
- [ ] `/warning/events` 能展示事件列表、风险等级、动态/静态/综合风险。
- [ ] `/warning/events/[id]` 能展示事件详情、风险解释、闭环模板、处置建议。
- [ ] `/source-tracing/attention` 能展示贡献节点、R/B 来源、业务边界。
- [ ] `/source-tracing/events/[id]` 能展示事件流转和台账。
- [ ] `/data/dynamic` 能展示动态数据趋势和近期采样，不全量加载。
- [ ] `/data/static` 能展示静态数据、静态风险试算和 S 项边界。
- [ ] `/data/features` 能展示 63 维动态通道口径、R/B 边界和当前后端节点数。
- [ ] `/model` 能展示手动预测入口，并提示可能写入台账。
- [ ] `/model/evaluation` 能展示 batch dry-run，不写正式台账。
- [ ] `/model/version` 能展示 config 只读快照并脱敏。
- [ ] `/system/config` 不能开放真实配置写入。
- [ ] 接口覆盖矩阵能展示 GET/POST/禁止接口状态。
- [ ] 页面不能把 B01-B41 写成真实传感器。
- [ ] 页面不能写 B 指标可单独断电、撤人或确认重大隐患。
- [ ] 页面不能把旧 62 维或旧固定通道数量作为当前事实。
- [ ] 页面不能出现浏览器直连 FastAPI 地址。
- [ ] `npm run lint` 和 `npm run build` 通过。

## 7. 数据边界验收

- [ ] 预警后端、动态数据、静态数据、事件台账、贡献解释、配置只读快照标注为真实后端接入。
- [ ] 双控、部分系统管理、真实通知、真实权限认证仍明确为当前未完成范围或 mock/只读边界。
- [ ] 生成/估计指标只作为辅助前兆和解释线索。
- [ ] S01-S32 静态风险项不占动态通道槽位。
