# 前端视觉与 UI 规则

## 1. 文件定位

本文件是当前前端视觉与 UI 规则唯一来源。阶段 9 视觉精修作为历史阶段保留；当前阶段只沿用本文件的视觉规则，不重新改 UI 风格。

不在本文件维护路由和页面业务最低需求：

- 路由清单：`docs/frontend-route-map.md`
- 页面业务最低需求：`docs/frontend-page-requirements.md`
- 长期状态：`docs/frontend-progress.md`

## 2. 当前主风格

当前主风格为：

```txt
Deep Mine Intelligence Console
深矿智能安全驾驶舱
```

Warm Safety Console 只作为历史阶段名保留，不再作为当前执行风格。

当前 UI 允许采用深色工业驾驶舱。禁止的是空洞炫酷大屏，不是深色界面本身。深色页面必须承载清晰业务：风险态势、传感器信息、阈值、待办处置、负责人、处理进度、更新时间和 mock 边界。

## 3. 历史阶段 9 视觉目标

阶段 9 已作为历史视觉精修阶段保留，其目标曾包括：

1. 参考图接近度提升。
2. PSD / 预览图组件质感转译。
3. 生产化信息表达增强。
4. 全站卡片、图表、标题条、底部状态条统一。
5. 重点页面视觉精修。

不重新规划导航、入口和路由层级。

## 4. 总览页业务原则

总览页第一屏必须优先回答：

- 现在安全吗？
- 风险在哪里？
- 哪些传感器触发？
- 当前值和阈值是多少？
- 谁在处理？
- 处理到哪一步？
- 下一步点哪里？

避免只有氛围、背景和图表，没有明确业务判断。

## 5. 视觉语言

核心关键词：

- 深蓝工业底色。
- 青蓝科技线条。
- 橙红风险强调。
- 玻璃拟态舱体卡片。
- 左中右驾驶舱结构。
- 中心煤矿生产态势主视觉。
- 顶部指标卡。
- 底部状态条或告警 ticker。
- 低透明粒子和慢速风险脉冲。

禁止：

- 空洞炫酷大屏。
- 高饱和霓虹堆叠。
- 大面积黑底红字。
- 强扫描线遮挡内容。
- 为了高级感牺牲业务可读性。

## 6. 指标卡标准

当前截图中的顶部指标卡样式保留为全站 `MetricCard` / `GlowMetricCard` 标准：

- 适合 3～5 个核心指标横向展示。
- 必须包含指标名称、数值、单位、状态或趋势。
- 关键指标应有微弱发光边框、角标、状态点或图标圆环。
- 风险类指标必须使用统一风险色。
- 指标卡不能只显示大数字，必须说明业务含义和更新时间或统计口径。

## 7. 风险颜色

全站统一：

- 低风险：蓝色
- 一般风险：黄色
- 较大风险：橙色
- 重大风险：红色

颜色用于风险徽章、指标趋势、热力点、图表序列和状态提示。不要为单页重新发明风险色。

## 8. PSD / 预览图转译规则

PSD、预览图和参考图只提取视觉语言，不复制代码、图片、品牌资产、完整布局或模板文案。

转译重点：

- 边框层级。
- 舱体卡片。
- 标题条。
- 指标卡。
- 图表密度。
- 状态徽章。
- 中心态势构图。
- 列表和详情的信息密度。

不要整图贴图，不要把 PSD 当背景糊到页面里。

## 9. PSD 组件映射

- 01：顶部导航、标题条、tab、状态按钮。
- 02：中心态势、数字孪生、动态通道结构。
- 03：详情抽屉、事件详情、隐患详情、传感器详情。
- 04：指标卡、风险徽章、状态徽章。
- 05：搜索、筛选、列表、表格。
- 06：排行、柱状图、趋势图、模型评估。
- 07：注意力权重、致因链路、知识图谱、雷达图。
- 08 / 09：巷道态势、风险热力、传感器点位、区域风险图。

## 10. 中心主视觉煤矿化

中心主视觉应更像煤矿生产态势，而不是普通装饰图：

- 巷道。
- 采掘面。
- 回风巷。
- 进风巷。
- 通风方向。
- 传感器点位。
- 风险标签。
- 区域状态。
- 阈值或异常提示。

无法做真实三维时，用简化拓扑、线框、热力、节点和标签表达。

## 11. 布局结构

总览页优先采用：

```txt
Topbar / ModuleHeader
MetricStrip
左洞察卡片 / 中心主视觉 / 右洞察卡片
底部状态条 / 快捷入口 / 告警 ticker
```

工作台页面可以降低装饰强度，但仍要保持标题条、筛选、表格、详情抽屉和状态徽章一致。

## 12. 系统管理低装饰规则

系统管理、日志、配置页可读性优先：

- 降低粒子、光效、透明叠层。
- 表格、权限矩阵、日志列表和配置项优先。
- 保留统一标题条、状态徽章、风险色和轻量边框。
- 不做复杂中心大屏。

## 13. 动效与截图验证

粒子、光效和动画必须服从可读性。

重大视觉改动只截图受影响页面，检查：

- 是否拥挤。
- 是否重叠。
- 文字是否不可读。
- 光效或粒子是否遮挡内容。
- 移动端是否横向溢出。

不再要求每次全站截图、全站评分或多轮视觉审计。
