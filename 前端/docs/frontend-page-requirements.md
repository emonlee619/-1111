# 前端页面最低需求

## 1. 文件定位

本文件是前端页面最低验收要求，用于补齐 `docs/frontend-route-map.md` 中全部路由的页面细节，避免后续 Codex / AI Agent 自由发挥。

本文件不替代：

- `docs/frontend-route-map.md`：路由树与一级模块边界来源。
- `docs/frontend-design-guide.md`：视觉风格、布局系统、阶段计划与核心设计来源。
- `docs/frontend-acceptance-checklist.md`：验收检查来源。

后续源码开发时，每完成一个路由骨架，都必须回查本文件，确认页面定位、最低展示内容、数据来源要求和禁止事项均已满足。

## 2. 路由根规则

- 只能使用 `app` 或 `src/app` 一套路由根。
- 如果选择 `app`，所有路由均放在 `app` 下。
- 如果选择 `src/app`，所有路由均整体平移到 `src/app` 下。
- 禁止同时存在 `app` 和 `src/app` 两套路由根。
- 源码阶段开始前必须先确认实际采用哪套路由根，并写入 `docs/frontend-progress.md`。

## 3. 全局数据边界

- 当前预警闭环页面允许通过 `/api/outburst/*` 接入真实后端只读能力和受控手动动作。
- 双控、部分系统管理、真实通知、真实权限认证仍为当前未完成范围，继续保持 mock、只读或禁用边界。
- 浏览器页面不得直连 FastAPI、数据库地址或模型文件路径；前端访问预警后端必须走 `frontend/src/lib/outburstApi.ts`。
- Mock 数据不得伪装成真实在线监测结论；真实后端数据、后端不可用占位、mock 演示和禁止开放能力必须区分展示。
- 当前通道口径为 63 维动态通道：R01-R22 真实传感器指标位 + B01-B41 物理约束生成/估计指标；页面节点数量以后端 meta/sensors/stats 实测返回为准。
- S01-S32 为静态风险项，不占动态通道槽位。

## 4. 全局禁止事项

- 禁止新增未在 `docs/frontend-route-map.md` 中确认的业务路由。
- 禁止在页面骨架阶段创建真实计算、真实文件上传、生产写入或后台任务入口。
- 禁止把 `/data/augmentation` 写成后台计算或生产任务页面。
- 禁止把复盘追溯写入隐患治理八步闭环。
- 禁止把八步闭环改写为其他顺序。

## 5. 模块总览与二级入口要求

- 每个一级模块必须有总览页，一级导航点击后默认进入该模块总览页。
- 模块总览页必须图表优先、主视觉优先，优先使用主视觉大面板、指标卡、排行、趋势、状态分布和快捷入口。
- 阶段 9 总览页必须回答：现在安全吗？风险在哪里？哪些传感器触发？当前值和阈值是多少？谁在处理？处理到哪一步？下一步点哪里？
- 表格和表单不得作为模块总览页主体；如需展示列表，只能作为辅助摘要或待办预览。
- 二级 tab 数量一般不超过 5 个，只承载业务大类，不承载所有细节功能。
- 每个二级 tab 顶部必须有 3～4 个小指标，用于说明当前视角的数量、比例、状态或时效。
- 详情页、事件详情、台账详情、配置页和专项工具页不作为二级 tab，应通过列表点击、卡片入口、详情抽屉或独立详情路由进入。
- 配置页必须通过右上角配置按钮或快捷入口进入，不得作为模块主路径。
- 既有细节路由可以保留为业务落点，但导航呈现必须服从 `docs/frontend-design-guide.md` 的二级入口合并规则。

双重预防二级入口固定合并为：

- 总览
- 风险管控
- 隐患治理
- 闭环复盘

双重预防配置不作为二级 tab，应放在模块右上角“配置”按钮中。

双重预防细节归属：

- 风险管控合并风险分级、风险四色图、风险告知卡、管控措施库。
- 隐患治理合并隐患排查、隐患台账、治理状态、逾期升级。
- 闭环复盘合并八步闭环、整改过程、验收审查、复盘追溯。

## 6. 数据增强展示边界

`/data/augmentation` 只展示数据增强结果、模拟指标和辅助说明：

- 不接真实后端计算任务。
- 不上传真实文件。
- 不写入生产数据。
- 不把生成类通道写成真实传感器监测值。
- 必须提示：生成类通道为辅助前兆指标，不替代真实传感器监测。

## 7. 八步闭环要求

隐患排查治理八步闭环必须是：

整理 → 分析 → 通报 → 整改 → 反馈 → 验收 → 审查 → 销号

- `/double-prevention/hazard-workflow` 必须严格展示上述八步。
- 复盘必须独立在 `/double-prevention/review`。
- 禁止把“复盘”加入八步闭环。
- 禁止写成“整理 → 分析 → 通报 → 整改 → 验收 → 审查 → 销号 → 复盘”。

## 8. 路由页面最低需求

### `/`

页面定位：
- 系统入口页，用于进入平台总览。

最低展示内容：
- 平台名称、当前阶段提示、进入 `/dashboard` 的主入口。
- 可展示简要安全态势摘要，但不承载复杂业务逻辑。

Mock 数据要求：
- 使用静态平台摘要、演示矿井名称和模拟更新时间。

禁止事项：
- 禁止在根路由堆叠完整业务大屏。
- 禁止接真实登录、真实后端或真实告警服务。

### `/dashboard`

页面定位：
- 综合驾驶舱，汇总矿井安全态势与重点待办。

最低展示内容：
- 当前风险等级、实时预警数量、重大隐患数量、双控闭环完成率。
- 重点区域风险排行、近期预警趋势、待办处置任务、数据更新时间。

Mock 数据要求：
- 使用 mock 指标卡、mock 趋势序列、mock 待办列表和 mock 区域排行。

禁止事项：
- 禁止把 dashboard 做成空洞炫酷大屏。允许采用 Deep Mine Intelligence Console 深色工业驾驶舱风格，但必须保留风险态势、传感器信息、待办处置、数据更新时间和 mock 边界。
- 禁止接入真实监测流或真实告警接口。

### `/monitoring`

页面定位：
- 实时监测与预警的监测侧入口。

最低展示内容：
- 监测模块概览、通道健康摘要、实时监测入口、通道管理入口。
- 最近异常波动提示和数据更新时间。

数据来源要求：
- 使用 `/api/outburst/health`、`/api/outburst/stats` 展示后端健康、动态数据量、实测节点数和最新数据时间；后端不可用时显示占位。

禁止事项：
- 禁止直接订阅真实传感器流。
- 禁止把生成通道混写为真实监测通道。

### `/monitoring/realtime`

页面定位：
- 传感器实时监测页。

最低展示内容：
- 后端实测监测节点、关键指标曲线、通道健康状态、最新采样时间、异常波动提示；通道数量以后端 meta/sensors/stats 返回为准。

数据来源要求：
- 使用 `/api/outburst/sensors/latest`、`/api/outburst/sensor-data/series`、`/api/outburst/sensor-data/recent`，不浏览器直连后端。

禁止事项：
- 禁止连接真实 WebSocket、MQTT、SSE 或生产监测接口。
- 禁止展示未标识来源的真实生产数据。

### `/monitoring/channels`

页面定位：
- 监测通道配置与健康巡检页。

最低展示内容：
- 通道列表、通道类型、所属区域、在线状态、最近更新时间、校验状态、维护负责人。

数据来源要求：
- 使用 `/api/outburst/meta` 和 `/api/outburst/sensors/latest` 展示通道元数据、单位、空间位置、source_type 和最新值。

禁止事项：
- 禁止提供真实启停、删除、校准传感器的操作。
- 禁止写入真实设备配置。

### `/warning`

页面定位：
- 预警事件模块入口。

最低展示内容：
- 预警总览、风险等级分布、待处置数量、事件列表入口、最新高风险事件摘要。

数据来源要求：
- 使用 `/api/outburst/warnings`、`/api/outburst/warnings/latest` 展示真实后端预警事件摘要；真实通知仍不开放。

禁止事项：
- 禁止触发真实预警通知。
- 禁止接生产事件接口。

### `/warning/events`

页面定位：
- 预警事件列表页。

最低展示内容：
- 事件编号、风险等级、事件时间、区域、关联通道、处置状态、负责人、筛选与检索。

数据来源要求：
- 使用 `/api/outburst/warnings` 展示事件列表、风险等级、动态/静态/综合风险和处置状态。

禁止事项：
- 禁止修改真实事件状态。
- 禁止发送真实短信、电话、企业微信或邮件通知。

### `/warning/events/[id]`

页面定位：
- 单个预警事件详情页。

最低展示内容：
- 事件基础信息、风险等级、关联区域、关联通道曲线、处置进度、溯源入口、处置建议。

数据来源要求：
- 使用 `/api/outburst/warnings/{id}`、`/api/outburst/warnings/{id}/contribution`，并展示风险解释、闭环模板和处置建议。

禁止事项：
- 禁止执行真实确认、关闭或升级预警操作。
- 禁止把 AI 建议写成强制安全结论。

### `/source-tracing`

页面定位：
- 预警溯源分析总览页。

最低展示内容：
- 最近预警的注意力权重、主要贡献指标、疑似危险源、致因链路、事件详情跳转。

数据来源要求：
- 使用预警后端贡献解释和事件台账数据；贡献度只作为模型解释线索，不写成最终事故原因。

禁止事项：
- 禁止调用真实模型推理。
- 禁止把溯源结果写成未经复核的最终事故原因。

### `/source-tracing/attention`

页面定位：
- 注意力权重与指标贡献解释页。

最低展示内容：
- 指标贡献排行、时间窗口、关联传感器、贡献变化趋势、解释说明。

数据来源要求：
- 使用 `/api/outburst/warnings/{id}/contribution` 和 `/api/outburst/meta`；R/B 来源、业务边界必须可见。

禁止事项：
- 禁止实时调用模型解释接口。
- 禁止隐藏“辅助解释”边界。

### `/source-tracing/events/[id]`

页面定位：
- 指定事件的溯源详情页。

最低展示内容：
- 事件摘要、注意力权重、危险源定位、致因链路、关联隐患、回到预警详情入口。

数据来源要求：
- 使用 `/api/outburst/events/ledger` 和事件详情；后端缺少阶段字段时保守展示，不编造完成状态。

禁止事项：
- 禁止将 mock 溯源结论写入真实事件台账。
- 禁止接真实模型服务。

### `/regions`

页面定位：
- 区域风险中心列表页。

最低展示内容：
- 区域名称、风险等级、隐患数量、预警数量、最近更新时间、管控状态、详情入口。

Mock 数据要求：
- 使用 mock 区域清单、mock 风险等级和 mock 管控状态。

禁止事项：
- 禁止读取真实矿图或真实区域数据库。
- 禁止提供真实区域配置写入。

### `/regions/[regionId]`

页面定位：
- 区域风险与隐患详情页。

最低展示内容：
- 区域基础信息、当前风险等级、传感器分布、关联隐患、管控措施、预警历史、闭环进度。

Mock 数据要求：
- 使用按 `regionId` 匹配的 mock 区域详情、mock 传感器点位和 mock 隐患列表。

禁止事项：
- 禁止展示未脱敏真实空间信息。
- 禁止修改真实管控措施。
