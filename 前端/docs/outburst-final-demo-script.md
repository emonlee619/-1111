# 瓦斯突出预警后端展示闭环最终演示脚本

## 1. 演示前准备

1. 启动 alert warning FastAPI 后端。
2. 在前端环境中设置 `OUTBURST_API_BASE_URL` 指向预警后端。
3. 启动前端：`cd frontend && npm run dev`。
4. 访问 `/api/outburst/health`，确认代理返回健康状态。

## 2. 演示路线

### 第一步：`/monitoring`

说明后端健康状态、动态数据量、当前后端返回节点数和最新动态数据时间。

### 第二步：`/monitoring/realtime`

说明实时监测、趋势曲线、近期采样表；后端不可用时页面显示占位，不伪装真实数据。

### 第三步：`/monitoring/channels`

说明通道元数据、单位、空间位置和来源；节点数量以后端返回为准。

### 第四步：`/warning/events`

说明预警事件列表、风险等级、动态风险、静态风险、综合风险和处置状态。

### 第五步：`/warning/events/[id]`

说明事件详情、风险组成、贡献节点、闭环模板、处置建议和人工复核提示。

### 第六步：`/source-tracing/attention`

说明贡献节点、R/B 来源边界和业务含义；贡献度是模型解释线索，不等于唯一物理因果。

### 第七步：`/data/features`

说明 63 维动态通道：R01-R22 真实传感器指标位，B01-B41 物理约束生成/估计指标。

### 第八步：`/data/static`

说明 S01-S32 静态风险项、静态风险试算边界和“不占动态通道槽位”。

### 第九步：`/model/evaluation`

说明 batch dry-run 默认不写正式台账；如果含生成/估计通道，只作为辅助训练/解释线索。

### 第十步：`/model/version` 和 `/system/config`

说明配置只读、敏感字段脱敏、真实配置写入禁用。

## 3. 必须强调的三句话

- 当前节点数量以后端 meta/sensors/stats 实测返回为准，不硬编码。
- B01-B41 是物理约束生成/估计指标，只能辅助前兆解释，不能替代真实监测。
- `PUT config`、`DELETE`、自动写库和未鉴权配置写入没有开放。

## 4. 常见问题回答

### 为什么不是旧固定通道数量？

旧固定通道数量属于历史方案。当前以 63 维附录卡为动态通道口径，并以后端 meta/sensors/stats 返回的实测节点数量展示页面状态。

### 为什么 63 维不等于页面固定显示 63 个节点？

63 维是模型动态输入特征口径；页面节点数是后端当前返回的 meta、sensors 或 stats 结果。后端数据缺失、筛选或版本变化时，页面不能硬编码节点数量。

### B 指标能不能作为撤人依据？

不能。B01-B41 是物理约束生成/估计指标，只能作为辅助前兆和待复核线索。撤人必须结合真实传感器、现场复核、调度处置和矿井应急预案。

### 静态风险项 S01-S32 是否属于动态通道？

不属于。S01-S32 是静态风险项，用于风险先验、管控优先级和隐患整改闭环，不占 R/B 动态通道槽位。

### 后端不可用时页面怎么处理？

页面通过 `/api/outburst/*` 代理获取数据。后端不可用时显示不可用占位和重试入口，不把空结果伪装成真实生产状态。

### 哪些页面是真实后端，哪些仍是 mock？

预警后端、动态数据、静态数据、事件台账、贡献解释、配置只读快照属于真实后端接入。双控、部分系统管理、真实通知、真实权限认证仍不是当前完成范围，继续保持 mock、只读或禁用边界。
